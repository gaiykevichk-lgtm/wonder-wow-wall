## ТОП критических пробелов

### 1. Подписочная модель не совпадает с ТЗ
- **ТЗ:** тарифы по площади — 15м²/7 000₽, 30м²/12 000₽, 50м²/18 000₽
- **Реализация:** тарифы по количеству накладок — 10шт/4 900₽, 25шт/9 900₽, безлимит/19 900₽
- **Действие:** уточнить с заказчиком актуальную модель

### 2. PostgreSQL не подключён
- Бэкенд работает на in-memory репозиториях — **данные теряются при рестарте**
- ORM-модели (10 таблиц) описаны в `models.py`, но не используются
- Alembic настроен, но **миграции не созданы**
- **Действие:** создать Alembic-миграции, реализовать SQL-репозитории, подключить к PostgreSQL

### 3. Визуализатор: нет бэкенда
- `domain/visualizer/` **полностью отсутствует** на сервере
- ML-сегментация на фронте — мок (белая маска)
- Авторазмещение панелей не работает
- Сохранение проектов визуализации — заглушка
- **Действие:** реализовать бэкенд-домен visualizer, подключить ML-модель или внешний API

### 4. Блог полностью отсутствует
- В ТЗ: «статьи, тренды, кейсы»

- **Действие:** решить приоритет; если нужен в MVP — реализовать домен content/blog

### 5. Социальная авторизация (Google, VK)
- Нет ни на фронте, ни на бэкенде — ни кнопок, ни OAuth-flow, ни плейсхолдеров
- **Действие:** как минимум добавить неактивные кнопки с текстом "Скоро"

### 6. Инфраструктура не настроена
- **Nginx** — нет reverse proxy
- **CI/CD** — нет автодеплоя
- **SSL/TLS** — нет HTTPS
- **S3** — нет хранилища файлов
- **CSRF-защита** — отсутствует
- **Rate limiting** — отсутствует

- **Действие:** необходимо для продакшна, но допустимо для MVP

### 7. Отзывы на странице товара не подключены к бэкенду
- Бэкенд: `GET /api/designs/{id}/reviews` + `POST /api/designs/{id}/reviews` — **готовы**
- Фронтенд: показывает только агрегированный рейтинг, **нет списка отзывов и формы добавления**
- **Действие:** подключить фронтенд к существующим эндпоинтам

- Реализованы: категория, цена

- **Действие:** добавить фильтры или уточнить приоритет
